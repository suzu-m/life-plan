# 「支出：車」画面の実装計画

自動車の購入および維持にかかる費用をシミュレーションするための「支出：車」画面（`/expense/car`）を新しく構築し、入力データを管理するストアを作成します。

## 概要

1. **ストアの作成 (`src/store/useCarStore.ts`)**
   - 複数台の車を管理できるようにします（車の台数に対応）。
   - 各車ごとのローン、残クレ、車検、税金などのデータを保持します。
2. **UI画面の構築 (`src/app/expense/Car.tsx`)**
   - 家族構成（Family.tsx）と同じように、車を追加・編集・削除できるインターフェースを提供します。
   - 車ごとにローン回数、金額、残クレ、車検、税金などを設定できる入力フォームを作成します。

## データ構造 (`CarPlan`)

以下のようなデータ型を想定し、Zustand の `persist` を使ってローカルストレージに永続化させます。

```typescript
export type CarPlan = {
  id: number;
  name: string; // 車の名前（例: 自家用車、夫の通勤用 など）
  purchaseYear: number | null; // 購入年

  // ローン関連
  loanPayments: number | null; // ローンの回数（月数。例: 60回）
  loanMonthlyAmount: number | null; // 月々のローン額（万円）
  
  // 残クレ関連
  hasZankure: boolean; // 残価設定型クレジットかどうか
  zankureFinalAmount: number | null; // 残価（最終回お支払い額 / 万円）
  
  // 維持費関連
  isNewCar: boolean; // 新車かどうか（新車=初回車検は3年後、以降2年毎。中古車=2年毎 等の計算用）
  shakenAmount: number | null; // 車検費用（1回あたり / 万円）
  taxYearlyAmount: number | null; // 自動車税（年額 / 万円）
  maintenanceYearlyAmount: number | null; // ガソリン・保険などその他維持費（年額 / 万円）
}
```

## UI コンポーネント (`Car.tsx`)

1. **車の一覧と追加機能**
   - 画面を開くと現在登録されている車のリストカードが表示されます。
   - 「車を追加する」ボタン（または下書きフォームからの追加）で新しい車を登録できます。
   
2. **各車の入力項目**
   - **車の情報**: 車の名称、購入年
   - **ローン情報**:
     - `NumberField` を用いて「支払い回数（回）」と「毎月の支払い額（万円）」を入力。
     - 「残クレを利用する」チェックボックスまたはスイッチ。
     - 残クレONの場合のみ、「残価設定額＝最終回に払う金額（万円）」の入力欄を表示。
   - **維持費**:
     - 新車／中古車 の選択（車検の発生タイミング計算のため）。
     - `NumberField` で「車検代（万円）」および「自動車税（年額 / 万円）」を入力。
     - あわせて利便性のため「その他維持費（保険代・ガソリン代など年額 / 万円）」の入力枠も設けます。
