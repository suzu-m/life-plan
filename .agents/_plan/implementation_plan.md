# 子供の支出を結果（グラフ）に反映させる実装計画

現在、`/results` 画面のグラフには「住宅費」のみが反映されています。
ここで入力された「子供の教育費（学費・習い事）」や「ライフイベント（結婚祝い等）」の支出をシミュレーションに乗せ、合算して表示できるように拡張します。

## 概要

1. 子供の年齢から生年を逆算し、カレンダー上の「年（Year）」ごとの支出を計算するマッピングを作成します。
2. 計算された各年の子供関連の支出（万円単位で入力されているため、円単位に換算）をグラフのデータポイントに追加します。
3. グラフの表示期間（開始年・終了年）を、住宅費の期間だけでなく、子供の教育期間（またはライフイベントの最終年）もカバーできるように拡張します。

## 対応内容

### 1. `Results.tsx` への計算ロジック追加

`src/app/Results.tsx` に以下の変更を加えます。

- `useChildStore` と `useFamilyStore` から子供の情報を取得。
- 子供ひとりひとりの「その年における年齢（`ageInYear`）」を計算し、年齢に応じて各期間の学費と習い事費用を加算する `calculateChildExpenseForYear` 関数を作成。
  - **保育園 (Nursery)**: 0〜2歳、3〜5歳で費用を切り替え。開始年齢でのフィルターも考慮。
  - **幼稚園 (Kindergarten)**: 開始年齢（デフォルト3）〜5歳の期間で費用計算。
  - **小学校・中学校・高校**: それぞれ 6〜11、12〜14、15〜17歳として費用を計算。
  - **高等教育**: 18歳から `higherEducationDuration` で指定された期間（年数）分だけ費用を計算。
  - **ライフイベント**: `event.age === ageInYear` の場合に費用を加算。
  ※金額はすべて「入力値（万円）× 10,000」で円単位に変換。

### 2. チャート表示期間の拡張

現在 `buildHousingChartData` で `plans` (住宅費) から `startYear` / `endYear` を割り出していますが、これを拡張します。
子供が存在する場合、「現在年（`new Date().getFullYear()`）」から始まり、「一番末っ子の高等教育が終わる年」または「最後のライフイベントの年」までをカバーできるように `endYear` を補正します。

### 3. バーチャート（BarChart）の更新

- グラフの積み上げ（Bar）シリーズに「子供の支出」を追加します（色は例として `#f94144` のような暖色系を想定。MUIのパレットのトーンに合わせます）。
- 総支出（`total`）に住宅費だけでなく「子供費」も合算します。
