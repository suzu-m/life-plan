# 「支出：生活費」画面の実装計画

家族の毎月の基礎的な生活コストを入力するための画面（`/expense/living`）と、それを管理するストアを作成します。

## 概要

1. **ストアの作成 (`src/store/useLivingStore.ts`)**
   - 家族全体における毎月の生活費カテゴリ別のデータを保持します。
   - 車や住宅のような「複数（リスト）」ではなく、生活費は「1つのデータ（シングル）」として管理する想定で設計します。
2. **UI画面の構築 (`src/app/expense/Living.tsx`)**
   - 食費や水道光熱費など、各項目ごとに直感的に入力できるフォームを作成します。

## データ構造 (`LivingPlan`)

以下のようなデータ型を想定します。金額は他の画面と同様に「万円」単位での入力がわかりやすいかと思います。

```typescript
export type LivingPlan = {
  foodMonthlyAmount: number | null; // 食費（月額/万円）
  utilitiesMonthlyAmount: number | null; // 水道光熱費（月額/万円）
  telecomMonthlyAmount: number | null; // 通信費（月額/万円）
  insuranceMonthlyAmount: number | null; // 生命保険・医療保険など（月額/万円）
  hobbiesMonthlyAmount: number | null; // 趣味・娯楽・交際費（月額/万円）
  otherMonthlyAmount: number | null; // 日用品・その他（月額/万円）
}
```

## UI コンポーネント (`Living.tsx`)

- 直感的なフォームUI: `NumberField` コンポーネントを並べ、カテゴリ別に月額（万円）を入力できるようにします。
- 合計額の表示: 入力されたすべての金額を合算し、「生活費の合計：月額 ◯◯万円 / 年額 ◯◯万円」とリアルタイムにプレビュー表示して見やすくします。

## Open Questions

生活費のシミュレーションにおいて、以下のアプローチについてご希望はありますか？

1. **年齢による変動**：例えば「退職後（60歳以降など）は生活費を現役時代のXX%にする」といった設定項目は必要でしょうか？（ひとまず「一定額がずっと続く」シンプルなモデルにしておくことも可能です）
2. **項目の増減**：上記のカテゴリ以外に、「お小遣い」「被服費」なども細かく分けたい、あるいは「ざっくり生活費として1つに入力したい」などのご希望はありますか？
